* Legacy migration removal

* Service catalogue consolidation

* Startup cleanup
* Scheduler cleanup
* Navigation audit

The goal is to ensure there is:

* ONE source of truth
* ZERO migration debt
* ZERO legacy logic
* ZERO historical upgrade logic
* ZERO startup backfills
* ZERO startup repair operations

Treat the current architecture as the only valid architecture.

PHASE 2 — SCHEMA CONSOLIDATION
==============================

Audit every table, column, index, enum, and schema ensure.

Verify:

* no duplicate columns
* no obsolete columns
* no abandoned fields
* no historical sprint leftovers
* no shadow data structures

Remove anything no longer used.

======================================

====================================================
PHASE 4 — SEED SYSTEM AUDIT
===========================

Audit all seeds.

Verify:

* deterministic
* idempotent
* required

Remove:

* historical seeds
* upgrade seeds
* repair seeds
* conversion seeds

Only bootstrap/catalog seeds should remain.

====================================================
PHASE 5 — RUNTIME LOG AUDIT
===========================

Review all startup logs.

Remove:

* historical terminology
* migration terminology
* sprint terminology
* phase terminology
* redundant startup noise

Keep only:

* startup success
* startup warning
* startup failure
* scheduler health

Startup output should be concise and production-grade.
