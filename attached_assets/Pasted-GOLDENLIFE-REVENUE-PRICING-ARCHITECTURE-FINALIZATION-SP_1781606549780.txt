====================================================
FINAL GOLDENLIFE REVENUE RULES
==============================

RULE 1

Provider service prices are defined in provider currency.

Examples:

Provider Currency = HUF

Consultation = 5000 HUF

Home Visit Add-on = 2000 HUF

Video Add-on = 0 HUF

These values become the source of truth.

====================================================
RULE 2

ALL BOOKING CALCULATIONS MUST OCCUR IN BOOKING CURRENCY.

Forbidden:

Provider Currency
→ USD
→ Fee Calculation
→ Convert Back

Remove this everywhere.

Allowed:

Base Price
+
Visit Type Charges
+
Optional Add-ons
================

Subtotal

Commission
Platform Fees
Taxes
Discounts

All calculated in booking currency.

====================================================
RULE 3

REMOVE ALL FIXED USD FEES.

Audit entire codebase.

Find:

Fixed Platform Fees
Fixed Booking Fees
Fixed Processing Fees
Fixed Service Fees
Fixed Revenue Fees

Replace with:

Percentage-based calculations

Examples:

Commission %
Platform Fee %
Service Fee %
Processing Fee %

No customer-facing fee may depend on exchange-rate changes.

====================================================
RULE 4

HOME VISIT FEES

Must belong to the service.

Stored in provider currency.

Calculated in booking currency.

No USD conversion.

====================================================
RULE 5

VIDEO VISIT FEES

Must belong to the service.

Stored in provider currency.

Calculated in booking currency.

No USD conversion.

====================================================
RULE 6

BOOKING SNAPSHOT STANDARDIZATION

Every booking must permanently store:

base_price

home_visit_fee

video_fee

service_addons_total

subtotal

commission_percent
commission_amount

platform_fee_percent
platform_fee_amount

processing_fee_percent
processing_fee_amount

tax_percent
tax_amount

discount_amount

final_total

currency

booking_currency

provider_currency

patient_currency

exchange_rate_used

exchange_rate_timestamp

final_total_usd

pricing_breakdown

This becomes the authoritative financial record.

Once booking exists:

NEVER RECALCULATE.

NEVER MUTATE.

NEVER UPDATE HISTORICAL VALUES.

====================================================
RULE 7

USD BECOMES REPORTING CURRENCY ONLY.

USD is forbidden from:

Service Pricing

Membership Pricing

Package Pricing

Commission Calculation

Platform Fee Calculation

Booking Calculation

USD may only exist for:

Reporting

Analytics

Accounting

Financial Exports

Provider Revenue Reports

Executive Dashboards

====================================================
RULE 8

MEMBERSHIPS

Stop using live exchange-rate pricing.

Implement fixed local pricing.

Example:

Gold Membership

USD = 29
EUR = 27
GBP = 24
HUF = 10990
INR = 2490
AED = 109

Users always see configured local pricing.

Exchange-rate changes must not affect membership pricing.

====================================================
RULE 9

PACKAGES

Stop using live exchange-rate pricing.

Implement fixed local pricing.

Example:

Package A

USD = 100
EUR = 92
GBP = 85
HUF = 37990
INR = 8490
AED = 379

Users always see configured local pricing.

Exchange-rate changes must not affect package pricing.

====================================================
RULE 10

EXCHANGE RATE USAGE

Exchange rates may only be used for:

Display conversion

Analytics

Financial reporting

Accounting

Snapshot creation

Exchange rates must NEVER alter:

Service pricing

Booking pricing

Commission calculations

Package pricing

Membership pricing

Provider earnings already booked

====================================================
RULE 11

PROVIDER EARNINGS

Keep current snapshot architecture.

Provider earnings must originate from booking snapshots.

Never recompute earnings later using current exchange rates.

Never recompute historical commission amounts.

====================================================
RULE 12

REFUNDS

Refunds must use the original booking snapshot.

Not current exchange rates.

Not current pricing.

Not current commission structures.

Refund calculations must remain deterministic.

====================================================
RULE 13

PAYOUTS

Payouts must originate from locked provider earnings records.

Not dynamic recalculation.

Not current exchange rates.

Not current service pricing.
